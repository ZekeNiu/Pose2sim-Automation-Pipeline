# Pose2Sim 中文 GUI 使用说明

## 启动与目录

双击项目根目录下的 `launch_pose2sim_gui.bat` 启动。程序会使用：

`D:\Application\Anaconda\envs\sports3d\python.exe`

所有项目文件和输出文件都保存在：

`D:\Application\Biomechanics\Pose2sim_Pipeline`

用户视频、校准资料、运行结果和报告不会上传到 GitHub。普通用户不需要手写 `Config.toml`，GUI 会在运行前自动生成。

## 推荐流程

按 GUI 顶部顺序操作：

1. `环境`：检查 Pose2Sim、OpenSim、ffmpeg 和报告依赖；需要时点击“一键更新 Pose2Sim”。
2. `项目`：创建或打开一个项目。
3. `校准`：生成棋盘格，导入内参视频，选择并准备外参方法。
4. `视频`：导入每个机位的正式测试视频。
5. `参数`：填写身高、体重、同步动作大致时间，必要时调整模型模式。
6. `运行`：保存配置、运行 Pose2Sim、生成 HTML 和 Excel 报告。

每一步底部都有“下一步”按钮，适合按向导式流程完成。

## 相机和手机摆放

- 至少 2 个机位。单人动作推荐一个正面机位，一个 45° 侧前方机位。
- 相机高度尽量接近髋部，主体从头到脚全程可见。
- 两个机位之间要有明显视角差，不要几乎重合，也不要一个机位完全看不到关键关节。
- 光照稳定，背景简洁，避免旁人、三脚架、杂物被识别成人体。
- 手机横竖屏都可以导入；程序会用 ffmpeg 生成规范化 MP4，减少旋转元数据导致的视频处理错误。
- 正式动作开始前，建议录入一个明显同步动作，例如快速下蹲起立、纵向跳动或大幅挥手。

## 棋盘格打印

GUI 生成的棋盘格默认是 `4 x 7` 个内角点，方格边长默认 `35 mm`。

- 可以直接用 A4 打印。
- 打印时必须选择“实际大小”或“100%”，不要选择“适应页面”。
- 打印后用尺子量一个方格边长。
- 如果实测边长是 35 mm，GUI 参数不用改。
- 如果实测边长不是 35 mm，例如 34.6 mm，就把实测值填到“棋盘格方格边长(mm)”。

A4 棋盘格通常适合内参校准。若用棋盘格做外参，全身动作场景里 A4 往往偏小，优先推荐“场景点外参”或使用更大的棋盘格。

## 校准

校准分为内参和外参。内参描述镜头畸变；外参描述每台相机在空间中的位置和朝向。只要相机移动，外参就需要重做。

### 内参视频怎么录

每台相机都要单独录一段棋盘格视频。

要求：

- 使用正式拍摄时相同的分辨率、焦距、变焦、横竖屏方向和对焦方式。
- 每段约 20-40 秒。
- 拿着棋盘格缓慢移动，让棋盘格出现在画面中央、四角、近处、远处。
- 适当倾斜和旋转棋盘格，不要只在一个平面里平移。
- 多数画面里棋盘格要完整可见。
- 避免反光、运动模糊、遮挡、严重过曝或过暗。

录完后，在 `校准` 页点击“导入内参棋盘格视频”，选择每台相机对应的视频。

### 外参方法 1：场景点（推荐）

场景点外参通常更适合全身动作和手机多机位。

操作：

1. 正式摆好所有相机后，不要再移动相机。
2. 录制或截取每个机位都能看到的静态场景。
3. 选择真实空间中可测量的点，例如地面标记、地砖角点、墙角、箱体角点、跑台边缘。
4. 建议至少 10 个点，尽量覆盖运动区域，并尽可能包含不同高度。
5. 在 GUI 下方填写这些点的真实 3D 坐标，单位是米。

坐标示例：

```text
[[0, 0, 0], [1, 0, 0], [0, 0.3, 0], [0, 0, 0.5]]
```

这些是现实世界坐标，不是视频像素坐标。点越分散、越容易在各机位里准确点击，外参越可靠。

### 外参方法 2：棋盘格

如果没有条件测量场景点，可以选择棋盘格外参。

操作：

1. 正式摆好所有相机后，不要再移动相机。
2. 把棋盘格放在所有相机都能清楚看到的位置。
3. 可以水平放在地面，也可以垂直固定；在 GUI 的“棋盘格放置方式”里选择对应方式。
4. 录制 3-5 秒清晰视频，或导入清晰图片。
5. 棋盘格越大、越接近运动区域，外参越稳。

注意：A4 棋盘格适合内参，但用于全身动作外参时可能偏小。如果外参误差大，优先改用场景点或更大的棋盘格。

## 正式测试视频

在 `视频` 页导入每个机位的正式测试视频。

要求：

- 每个机位尽量同时开始录制。
- 主体完整可见，关键关节不要长期被遮挡。
- 正式动作前或动作中加入明显同步动作。
- 避免视频过短；建议至少 5 秒以上。
- 导入后 GUI 会保存原始文件，并生成规范化 MP4。

## 同步

手机多机位通常没有硬件同步，所以需要同步动作。

推荐同步动作：

- 明显纵向跳动。
- 快速下蹲再站起。
- 双臂快速上举或挥手。
- 清晰闪光事件也可以，但纯视觉闪光不一定适合所有关键点同步。

在 `参数` 页填写每个机位中同步动作的大致发生时间，单位是秒，例如：

```text
1.2, 1.4
```

这个时间不需要精确到帧。Pose2Sim 会在这个大致时间附近搜索最大速度或相关峰。只要填写值落在搜索范围内，通常不会影响结果。若不确定，可以在高级参数里适当增大“同步搜索范围”，但范围太大可能匹配到错误动作。

只有在相机已经通过硬件或同一设备严格同步时，才勾选“视频已硬同步，跳过同步”。

## 参数建议

基础参数：

- 身高：尽量填写真实身高；留空时会自动估计，但尺度和标记点增强可能更不稳定。
- 体重：对关节角度影响很小，主要影响后续动力学；未知时保留 70 kg。
- 姿态模型：默认“身体+足部”，适合大多数体能和康复动作。
- 模型模式：默认“平衡”。预览可用“更快”，最终分析可用“更准”。
- 保存叠加检测视频：默认开启，便于检查识别质量。
- 标记点增强：默认开启；如果条件不满足，运行时会自动跳过并记录原因。

高级参数：

- 足部贴地修正：适合站立、步态等地面接触明显的动作；跳跃腾空或非地面动作不要随意开启。
- 左右对称：普通分析保持开启；明显肢体不对称时再考虑关闭。
- 简化 OpenSim 模型：只在运行失败或需要快速预览时尝试。
- 滤波截止频率：普通体能动作保持 6 Hz。

## 输出

每个项目输出到：

`outputs/<项目名>/reports`

主要文件：

- `*_关节活动度.html`：交互式图表、同步视频、质量诊断摘要、完整诊断弹窗、统计表和指标解释。
- `*_关节活动度.xlsx`：逐帧关节活动度、统计摘要、指标定义、质量诊断。

报告中的角度来自 OpenSim `.mot` 逆运动学结果，不是原始 2D/3D 坐标。

## 解释边界

报告用于动作分析和质量检查，不构成医学诊断或康复处方。以下情况会降低可信度：

- 校准误差大。
- 同步动作不清楚或填写的大致时间偏离太多。
- 关键关节长期遮挡。
- 主体没有完整出现在多个机位中。
- 手机自动曝光、自动对焦或画面旋转导致视频质量不稳定。
- OpenSim marker error 较高。
