# Pose2Sim 中文 GUI 使用说明

## 启动与目录

双击项目根目录下的 `launch_pose2sim_gui.bat` 启动。程序使用：

`D:\Application\Anaconda\envs\sports3d\python.exe`

所有项目、规范化视频、Pose2Sim 中间结果和最终报告都保存在：

`D:\Application\Biomechanics\Pose2sim_Pipeline`

用户视频、校准资料和输出不会上传到 GitHub。普通用户不需要手写 `Config.toml`。

## 推荐主流程

默认服务“新建空白项目，从采集到报告一步步完成”的用户：

1. `环境`：检查 Pose2Sim、OpenSim、ffmpeg 和报告依赖；需要时一键更新 Pose2Sim。
2. `项目`：创建项目，默认选择 `单人`。多人属于高级用法。
3. `校准`：先准备内参和外参资料，再录制或导入正式动作视频。
4. `视频`：导入每个机位的正式测试视频，GUI 会规范化为浏览器兼容 MP4。
5. `参数`：填写身高、体重、同步动作大致时间和少量常用参数。
6. `运行`：保存配置并运行完整流程，或对已有项目按现有 Config 运行。

每页底部都有 `下一步` 和 `上一步`，适合按向导完成。

## 项目模式

### 新建项目

新项目默认是“单人、单次动作、多机位”。如果选择 `多人`，GUI 会写入 `multi_person=true`，身高和体重可以用逗号填写多个值，例如：

```text
1.80, 1.65
82, 60
```

多人分析更容易受遮挡、同步误差和人员匹配影响，建议先用单人流程确认相机布置和校准质量。

### 打开已有项目

打开别人分享的项目或官方示例时，GUI 会扫描通用结构和 `Config.toml`，识别：

- 空项目
- 已有视频但未分析项目
- 已有外部校准项目，例如 `calibration_type='convert'`
- 已分析项目，例如已有 `kinematics/*.mot`
- 多人项目
- 批处理项目

GUI 不按 `Demo_*` 名称做特殊适配。已有 `Config.toml` 默认不会被覆盖；如果用户选择重新生成配置，会先备份为 `Config.backup_YYYYMMDD_HHMMSS.toml`。

批处理属于高级用法。GUI 会把项目路径交给 Pose2Sim 官方层级配置读取逻辑，不会强行转换成普通单项目配置。

## 相机和手机摆放

- 至少 2 个机位。单人动作推荐正面 + 45° 侧前方。
- 相机高度尽量接近髋部，主体从头到脚全程可见。
- 机位之间要有明显视角差，避免几乎重合。
- 光照稳定，背景简洁，避免旁人和杂物被识别成人体。
- 手机横屏或竖屏都可以导入；GUI 会用 ffmpeg 规范化视频，减少旋转元数据导致的错误。
- 正式动作前建议加入明显同步动作，例如快速下蹲起立、纵向跳动或大幅挥手。

## 棋盘格打印

GUI 可以生成两种棋盘格：

- 内参棋盘格：默认 A4、`4 x 7` 内角点、`35 mm` 方格。
- 外参棋盘格：默认 A3、`4 x 7` 内角点、`45 mm` 方格。

打印时必须选择 `实际大小 / 100%`，不要选择“适应页面”。打印后用尺子测量一个方格边长，并检查图上的 `100 mm` 标尺。

如果实测方格尺寸与 GUI 中的参数一致，不需要改参数。如果不一致，必须把实测值填回 GUI。A4 通常适合内参；如果用棋盘格做全身动作外参，A3 是更稳的最低便利选择，能使用更大的硬质板更好。

## 校准

校准分为内参和外参。内参描述镜头畸变；外参描述每台相机在空间中的位置和朝向。相机一旦移动，外参需要重做。

### 内参视频怎么录

每台相机单独录一段棋盘格视频：

- 保持正式拍摄时相同的分辨率、焦距、变焦、横竖屏方向和对焦方式。
- 每段约 20-40 秒。
- 拿着棋盘格缓慢移动，让棋盘格覆盖画面中央、四角、近处、远处。
- 要有轻微倾斜和旋转，不要只在一个平面里平移。
- 多数画面里棋盘格要完整可见。
- 避免反光、运动模糊、遮挡、严重过曝或过暗。

录完后，在 `校准` 页点击 `导入内参棋盘格视频`，按机位顺序选择文件。

### 外参方法 1：场景点（推荐）

场景点外参通常更适合全身动作和手机多机位。

1. 先定义坐标系，例如运动区域地面中心为 `P1=[0,0,0]`，前方是 X 正方向，左方是 Y 正方向，上方是 Z 正方向，单位是米。
2. 3D 坐标来自现场测量：卷尺、激光测距仪、地砖尺寸、场地标线或已知物体尺寸都可以。
3. 点上建议放彩色胶带十字、编号贴纸、小锥桶、反光点。天然清晰角点也可以，例如墙角、箱体角点、跑台边缘。
4. 建议至少 10 个点，尽量覆盖运动区域周围，并包含不同高度。
5. 正式摆好所有相机后，不要再移动相机。拍每个机位都能看到这些点的外参资料：清晰图片即可，录 3-5 秒静态视频也可以。
6. 外参资料拍完后，只要相机不动，点位标记可以移走，再录正式动作视频。
7. 后续点选 2D 点时，点击顺序必须和 GUI 表格的 P1、P2、P3 一致。

GUI 中的场景点表格会自动转成 Pose2Sim 的 `object_coords_3d`。

### 外参方法 2：棋盘格

如果没有条件测量场景点，可以选择棋盘格外参：

1. 固定所有相机后，不要再移动相机。
2. 把较大的外参棋盘格放在所有相机都能清楚看到的位置。
3. 可水平放在地面，也可垂直固定；GUI 中的放置方式要和实际一致。
4. 录制 3-5 秒清晰视频，或导入清晰图片。
5. 内参棋盘格和外参棋盘格参数相互独立，实际打印尺寸不同就分别修改。

如果只能用 A4 做外参，可以尝试，但全身动作优先推荐场景点外参或更大的棋盘格。

## 同步

普通手机多机位通常没有硬件同步，所以需要同步动作。

推荐同步动作：

- 明显纵向跳动。
- 快速下蹲再站起。
- 双臂快速上举或挥手。

在 `参数` 页填写每个机位中同步动作的大致发生时间，单位秒，例如：

```text
1.2, 1.4
```

这个时间不需要精确到帧。Pose2Sim 会在该时间附近搜索最大速度或相关峰。只要填写值落在搜索范围内，通常不会明显影响结果。若不确定，可在高级参数中把“同步搜索范围”增大到 3-4 秒；范围过大可能匹配到错误动作。

只有相机已经通过硬件或同一设备严格同步时，才勾选“视频已硬同步，跳过同步”。

## 参数建议

基础参数：

- 身高：尽量填写真实值；多人用逗号分隔。
- 体重：对关节角度影响很小，主要影响后续动力学；未知时保留 70 kg。
- 姿态模型：默认 `身体+足部`，适合大多数下肢、步态、深蹲和体能动作。
- 模型模式：默认 `平衡`。预览可用 `更快`，最终分析可尝试 `更准`。
- 保存叠加检测视频：默认开启，便于检查遮挡、错人、左右混淆和低质量帧。
- 标记点增强：默认开启；如果模型或结果条件不满足，运行时会自动跳过并记录原因。

高级参数：

- 足部贴地修正：适合站立、步态等地面接触明显的动作；跳跃腾空或非地面动作不建议开启。
- 左右对称：普通分析保持开启；明显肢体不对称时再考虑关闭。
- 简化 OpenSim 模型：只在运行失败或需要快速预览时尝试。
- 滤波截止频率：普通体能动作建议保持 6 Hz。

## 输出

单项目输出在：

`outputs/<项目名>/reports`

主要文件：

- `*_关节活动度.html`：交互式图表、同步视频、质量诊断摘要、完整诊断弹窗、统计表和指标解释。
- `*_关节活动度.xlsx`：逐帧关节活动度、统计摘要、指标定义、质量诊断。

多人项目如果存在多个 `.mot`，HTML 会提供人员/结果选择，Excel 会按人员或结果分 sheet。批处理项目会为每个 Trial 生成报告，并在 `outputs/<项目名>/index.html` 生成入口页。

报告角度来自 OpenSim `.mot` 逆运动学结果，不是原始 2D/3D 坐标。

## 解释边界

报告用于动作分析和质量检查，不构成医学诊断或康复处方。以下情况会降低可信度：

- 校准误差大。
- 同步动作不清楚或大致时间偏离太多。
- 关键关节长期遮挡。
- 主体没有完整出现在多个机位中。
- 手机自动曝光、自动对焦或画面旋转导致视频质量不稳定。
- OpenSim marker error 较高。
