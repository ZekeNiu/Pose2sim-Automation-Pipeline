# Pose2Sim 中文 GUI 使用说明

## 启动

双击项目根目录下的 `launch_pose2sim_gui.bat`。

程序会使用：

`D:\Application\Anaconda\envs\sports3d\python.exe`

所有项目文件和输出文件都会保存在：

`D:\Application\Biomechanics\Pose2sim_Pipeline`

## 推荐拍摄

- 至少 2 个机位。
- 单人测试推荐：一个正面机位，一个 45 度侧前方机位。
- 相机高度尽量接近髋部，主体全程完整可见。
- 光照稳定，背景简洁，避免三脚架、旁人和杂物被识别成人体。
- 手机横竖屏都可以导入，程序会用 ffmpeg 生成规范化 MP4，减少旋转元数据造成的错误。

## 校准

### 内参

内参描述相机镜头和畸变。通常同一台相机、同一分辨率和焦段只需要做一次。

操作：

1. 在 GUI 生成棋盘格 PDF/PNG。
2. 100% 尺寸打印，并测量实际方格边长。
3. 每台相机拍一段移动棋盘格视频，让棋盘格出现在画面不同位置、距离和角度。
4. 在 GUI 导入每台相机的内参棋盘格视频。

### 外参

外参描述每台相机在空间中的位置和朝向。只要相机移动，就需要重新做。

默认推荐“场景点外参”：用地砖角点、墙角、箱体边缘、跑台边缘等可测量点，记录它们的 3D 坐标。点越分散越好，建议 10 个以上。

如果不方便测量场景点，可以选择“棋盘格外参”：把棋盘格放在地面或场景中，让所有机位都能看到。该方法更简单，但如果棋盘格太小或离运动区域太远，外参精度可能变差。

## 同步

手机通常没有硬件同步。建议正式动作前或动作中加入清晰同步事件：

- 明显纵向跳动、快速下蹲起立、挥手。
- 或使用清晰视觉事件，例如闪光灯。
- 采集长度至少 5 秒，避免视频过短。

如果知道每个机位同步动作发生的大致时间，在参数页填写秒数，例如：

`1.2, 1.4`

如果视频已经硬同步，可以勾选“视频已硬同步，跳过同步”。

## 输出

每个项目输出到 `outputs/<项目名>/reports`：

- `*_关节活动度.html`：交互式图表、同步视频、质量诊断、统计表和指标说明。
- `*_关节活动度.xlsx`：逐帧关节活动度、统计摘要、指标定义、质量诊断。

报告中的角度来自 OpenSim `.mot` 逆运动学结果，不是原始 2D/3D 坐标。

## 解释边界

报告用于动作分析和质量检查，不构成医学诊断或康复处方。以下情况会降低可信度：

- 校准误差大。
- 手机同步不清晰或动作事件不明显。
- 关键关节被遮挡。
- 人体没有完整出现在多个机位中。
- OpenSim marker error 较高。
